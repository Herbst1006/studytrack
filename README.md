# StudyTrack · 学习积累追踪器

> 让每一次学习行为都被看见——记录积累，而非追赶目标。

**[→ 查看 Demo](https://herbst1006.github.io/studytrack)**

---

## 项目简介

StudyTrack 是一款面向自主学习者的学习记录工具。

大多数学习类工具把「完成目标」作为衡量标准，但真实的学习往往没有固定终点——读一本书不一定有截止日，学一门语言没有满分线。StudyTrack 的出发点是：**记录本身就有价值**。当你能看见自己「在学 Python 这件事上已经投入了 31 小时」，这个积累的轨迹本身就是驱动力。

对于有明确目标的用户（备考、课程完成等），进度条仍然可以选择开启。两种模式并存，用户自己决定。

## 核心功能

- **学习项目管理** — 添加任意学习项目，时长或章节数均可记录，目标量选填
- **快速打卡** — 3步完成一次打卡，进度实时更新
- **双模式展示** — 有目标显示进度条，无目标显示累计轨迹
- **学习统计** — 本周/本月时长柱状图，各项目占比，免费提供（竞品普遍锁付费墙）
- **容错打卡机制** — 用「近30天活跃率」替代「连续打卡天数」，断签不清零

## 项目背景

这是一个从 0 到 1 的完整产品练习项目，覆盖产品经理的完整工作流：

```
用户研究 → 竞品分析 → 需求文档 → 原型设计 → Demo 开发 → 产品复盘
```

### 用户研究方法
通过 Python 脚本调用 Apple iTunes RSS 接口，采集 Forest、番茄ToDo、滴答清单、Notion 四款竞品的 App Store 评论，共获取 **1355 条原始数据**，分析 **326 条低分评论**，提炼 5 大用户痛点。

### 关键产品决策
项目过程中识别并推翻了一个早期假设——「强制填写目标量」会将大量没有明确学习终点的用户挡在门外。由此将产品定位从「进度追踪」重定向为「持续记录」，这个决策直接影响了核心功能设计。
